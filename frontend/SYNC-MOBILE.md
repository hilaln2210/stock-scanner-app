# סנכרון דסקטופ ↔ אפליקציית נייד (PWA)

**זו אותה אפליקציה** — בדסקטופ ובנייד (PWA). אין אפליקציית נייד נפרדת.

**רוצה שהנייד לא יהיה תלוי במחשב?** תפרוס את האפליקציה לשרת בענן (Render / Railway / VPS) — ראה **`docs/DEPLOY-MOBILE.md`**.

כדי שהנתונים בדסקטופ ובנייד יהיו מסונכרנים:

## מקור אמת אחד
כל הנתונים (תיק, סורק, חדשות, בריפינג) מגיעים **מאותו בקאנד**. דסקטופ ונייד משתמשים באותו API.

## איך זה עובד
- **בפיתוח:** ה־proxy של Vite מפנה `/api` ל־localhost:8000. הקובץ `.env` יכול להגדיר `VITE_API_URL=http://localhost:8000/api` אם צריך.
- **בפרודקשן:** מומלץ להגיש פרונט ובקאנד **מאותו דומיין** (למשל reverse proxy שמפנה `example.com` לפרונט ו־`example.com/api` לבקאנד). אז ברירת המחדל היא `baseURL: '/api'` — גם בדסקטופ וגם בנייד (PWA) משתמשים באותו בקאנד ולכן הנתונים מסונכרנים.
- **אם הפרונט והבקאנד על דומיינים שונים:** להגדיר בבניית הפרונט את אותה כתובת בקאנד לשניהם, למשל:
  ```bash
  VITE_API_URL=https://api.yourdomain.com/api npm run build
  ```

## רענון נתונים
- הופעל `refetchOnWindowFocus` ו־`refetchOnReconnect` — כשחוזרים לאפליקציה או לחיבור רשת, הנתונים מתעדכנים אוטומטית.

## PWA (התקנה בנייד)
- האפליקציה מוגדרת כ־PWA (Vite PWA). אפשר להוסיף למסך הבית מהדפדפן בנייד.
- וודא שבנייד הכתובת שפותחים היא **אותה כתובת** של האפליקציה (אותו שרת), כדי ששני המכשירים ישתמשו באותו API.
